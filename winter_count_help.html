Welcome to Winter Count

Winter Count is a comprehensive public health data system designed to support Tribal health authorities in managing communicable disease cases, improving data analysis, and ensuring Tribal data sovereignty. This guide provides essential resources to help you navigate and utilize the system effectively.

Key Features

1. Dashboard Overview

  • Central hub for data insights and navigation.
  • Reports & Visualizations: PDFs, static reports, Tableau dashboards, and Power BI reports.
  • Role-Based Access: Users see only data relevant to their permissions.

2. Case Management

  • Search cases by ID, name, or disease type.
  • Add, edit, and delete case notes.
  • View electronic lab reports (eLR) for real-time updates.

3. Data Analysis & Reporting

  • Create custom Power BI reports.
  • Access automated trend graphs for diseases like COVID-19, syphilis, and diabetes.
  • Training resources available for interpreting data.

4. Security & Compliance

  • RBAC: Role-based access control for confidentiality.
  • Audit Trails: Track system activity.
  • Secure Data Sharing: Tools for Tribal data sovereignty.

Getting Started

  • Login: Use your username and Duo authentication for secure access.
  • Explore the Dashboard: Review reports, search cases, and view real-time updates.

Fabric FAQ

What can I break in Fabric?

Users with Admin, Member, and Contributor roles can perform all CRUD operations in a Lakehouse. A Viewer can only read data via the SQL analytics endpoint. Do not delete items unless you are certain.

Learn more

Is Copilot AI enabled in Fabric?

Not until enterprise-level capacity (F64) is enabled.

What are naming conventions for Fabric?

Follow the format of current objects of similar type. Some object types have naming restrictions.

Can analysts connect to Fabric data from outside tools (e.g., SAS)?

Publicly available data can be accessed freely. For protected data, contact Informatics staff to move data to your folder in the RDS.

How can I share publicly available data externally?

Public data can be freely shared outside Fabric. For collaboration within Fabric with someone outside GPTLHB, contact Informatics & IT.

Can I export data from Fabric?

All data sources in Fabric can be exported; every action is logged. A workaround may be needed due to tenant differences.

How do I perform Exploratory Data Analysis (EDA)?

Use notebooks within Fabric (Python, R, SQL) to perform EDA. One benefit is not being limited by your computer’s memory.

Will my access change in Fabric?

No. You will have the same level of access to data sources. Restricted sources remain limited to authorized users.

Can I do no-code analysis (e.g., Excel pivot tables)?

CSV files can be analyzed in notebooks or loaded into delta tables and queried with SQL.
